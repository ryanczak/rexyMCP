# Bug 1 on phase-01: the spec'd `discount_rate_comes_from_architect_config` test was not written

**Severity:** minor
**Status:** verified
**Filed:** 2026-07-24

## What's wrong

The phase's § Test plan named four tests. Three were written; the fourth —
`discount_rate_comes_from_architect_config` in `mcp/src/costs.rs` — is absent.
`cargo test` totals are unchanged from before the phase (632 / 1032), confirming
no net new coverage in the `rexymcp` crate.

The production change is **correct** — reviewer verified
`254.00M in × $5 + 0.617M out × $25 = $1285.43`, matching `rexymcp costs`
exactly — but the behavior it establishes is unpinned:

```rust
// mcp/src/costs.rs:222 — the phase's entire point, untested
let (discount_in, discount_out) = cfg.architect.effective_rates();
```

The three tests that *were* written (`architect_effective_rates_from_model`,
`architect_effective_rates_explicit_override_when_model_unknown`,
`legacy_dashboard_section_is_ignored`) all exercise the `ArchitectConfig`
**accessor**. None asserts that `load_cost_report` actually *calls* it. Repoint
line 222 at any other rate source and the entire suite still passes.

That is the exact defect class this milestone exists to eliminate: a discount
rate sourced from somewhere nobody was checking. Shipping the fix without the
guard leaves the next refactor free to silently reintroduce it.

## What should happen

Per the phase doc's § Test plan:

> `discount_rate_comes_from_architect_config` — build a `Config` whose
> `[architect] model` is `claude-fable-5` (`$10/$50`), run the scope report over
> a fixture with known executor tokens, and assert `saved` reflects **$10/$50**,
> not `$5/$25`. This is the test that would have caught the original defect: it
> fails if anything still reads a separate dashboard rate.

`claude-fable-5` is the right fixture model precisely because its rates differ
from opus-4-8's — a test written against opus would pass under either source and
prove nothing.

## How to fix

Add the test to `mcp/src/costs.rs`'s `#[cfg(test)] mod tests`. The
`load_cost_report_telemetry_disabled_errors` test in the same module already
demonstrates the `TempDir` + written-`rexymcp.toml` fixture shape — reuse it
rather than inventing a new one, and give the config a `[telemetry]` dir plus a
`phase_runs.jsonl` carrying one `PhaseRun` with known `input_tokens` /
`output_tokens` so `saved` is a computable constant.

Asserting through `load_cost_report` is what makes the test meaningful. If that
proves impractical, assert through `scope_report` **plus** a direct assertion
that `load_cost_report` builds its `BudgetRates` from
`cfg.architect.effective_rates()` — but do not settle for re-testing the
accessor, which is already covered.

Add **only** this test. Do not modify the three existing new tests, the
production code, or any file outside `mcp/src/costs.rs`.
