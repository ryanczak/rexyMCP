# Bug 1 on phase-03: false_completion — self-reported complete on red gates

**Severity:** major
**Status:** verified
**Filed:** 2026-06-14
**Fixed:** 2026-06-15 (commit `0967f92`; re-verified at review — fmt/clippy clean)

## What's wrong

The executor self-reported `complete` without running `cargo fmt --all --check`
or `cargo clippy --all-targets --all-features -- -D warnings`. Both fail:

- `cargo fmt --all --check` exits 1 — `mcp/src/profile.rs` has formatting
  divergences (multi-line `sort_by` closure on lines 216–220 should be a
  one-liner; multi-line `make_review` call on lines 364–370 should be a
  single-line call).
- `cargo clippy -- -D warnings` exits 101 — 9 dead-code errors (covered by
  bug-03-2 below).

`cargo test` passes (8/8 new tests). The logic is correct; the gates were
simply not run before claiming done.

## What should happen

STANDARDS §1 and §4 require all four gates to pass before a phase is complete.
`cargo fmt --all --check` and `cargo clippy -- -D warnings` must exit 0.

## How to fix

1. Run `rustfmt mcp/src/profile.rs` to fix the two formatting divergences in
   `profile.rs` (the `sort_by` closure body and the `make_review` call in
   `strengths_come_from_folded_runs`).
2. Address the clippy dead-code errors per bug-03-2.
3. Re-run all four gates as separate invocations and confirm each exits 0 before
   reporting complete.
