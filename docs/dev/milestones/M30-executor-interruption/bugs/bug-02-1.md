# Bug 1 on phase-02: `get_run_status_running_times_out` blocks the test suite for a real 15 seconds

**Severity:** major
**Status:** open
**Filed:** 2026-07-10

## What's wrong

`get_run_status_running_times_out` (`mcp/src/server_tests.rs:977`) drives the
long-poll with the **production** window instead of a tiny test value:

```rust
let out = get_run_status_inner(&registry, &params, Duration::from_secs(15)).await;
assert_eq!(out.state, "running");
```

The run is inserted but never published, so `await_terminal` waits the full
`Duration::from_secs(15)` before returning `Running`. Measured independently:

```
$ cargo test -p rexymcp get_run_status_running_times_out
test result: ok. 1 passed; ... finished in 15.00s   (wall 15.10s)
```

The whole `rexymcp` suite now takes **15.04s** (was sub-second for this crate's
server tests), and every developer run and every CI run pays that 15-second tax.
This violates STANDARDS §3.3 ("Tests are **deterministic**: no `sleep`…") — a
15-second real wall-clock wait that always elapses is a `sleep` in all but name.

The sibling registry test in `mcp/src/jobs.rs:194`
(`await_terminal_times_out_to_running`) already does this correctly with
`Duration::from_millis(1)` — the two timeout tests are inconsistent.

## What should happen

The timeout-path test must exercise the "still running after the window"
branch **without** a multi-second real wait. STANDARDS §3.3: tests are fast and
deterministic. The assertion (`state == "running"`) does not depend on the
window being 15 s — only on the run never publishing, so any tiny timeout
elapses deterministically to `Running`.

## How to fix

In `mcp/src/server_tests.rs`, change `get_run_status_running_times_out`'s
timeout from `Duration::from_secs(15)` to `Duration::from_millis(1)` (mirror the
`await_terminal_times_out_to_running` test in `jobs.rs`). Do **not** change
`RUN_STATUS_POLL_TIMEOUT` (the production constant stays 15 s) — only the test's
injected timeout changes; that is exactly why `get_run_status_inner` takes the
timeout as a parameter.

- [ ] `cargo test -p rexymcp` total wall-clock drops back to a few seconds.
