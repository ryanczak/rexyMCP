# Bug 1 on phase-03: missing `Net` row not-parenthesized negative pin

**Severity:** minor
**Status:** resolved
**Filed:** 2026-06-30
**Resolved:** 2026-06-30 — `savings_lines_net_row_not_parenthesized` added (commit `1798d8a`), mutation-resistant; phase approved_after_1.

## What's wrong

The implementation is correct — `savings_lines` builds the `Net:` row via
`make_row` (`mcp/src/dashboard/panels.rs:597-612`), never `debit_row`, so Net is
never parenthesized. But the phase doc required a **test** pinning that behavior,
and it was not written. No existing test covers it:

- `savings_lines_net_subtracts_architect_from_baseline` (`panels.rs:1830`) asserts
  `net_line.contains("$4.00")`. A regression that parenthesized Net would render
  `($4.00)`, which **still contains** `$4.00` — so this test passes either way and
  does not pin the behavior.
- `savings_lines_data_rows_equal_width_for_alignment` (`panels.rs:1899`) checks
  fixed-column line widths. Parenthesization happens *inside* a `{:>9}`/`{:>10}`
  right-aligned column, so the line width is unchanged and this test would not
  catch it either.

The phase doc calls for this pin in three places:
- Spec § "Must NOT happen (pinned negatives — **write tests for these**)": "The
  `Net:` row must **never** be parenthesized, even when Architect is shown."
- Test plan: "`savings_lines_net_row_not_parenthesized` in `panels.rs` — with
  architect cost set so the Architect row shows, assert the `Net:` row contains
  no `(`."
- Acceptance criteria: "…and a `Net:` row with no parentheses."

## What should happen

A mutation-resistant test pins that the `Net:` row is never parenthesized, so a
future change that accidentally routes Net through `debit_row` is caught by the
suite. (The behavior under test is already correct; this only adds the missing
guard.)

## How to fix

Add `savings_lines_net_row_not_parenthesized` to the `#[cfg(test)] mod tests`
block in `mcp/src/dashboard/panels.rs`. Build a summary with session metrics and
non-zero architect cost (so the Architect row shows, proving parenthesization is
active in the same call), locate the `Net:` row by label substring (not index),
and assert it contains no `(`. For example, reuse the setup from
`savings_lines_net_subtracts_architect_from_baseline` and assert
`!net_line.contains('(')`.

Append a new Update Log entry; commit as `test:` (behavior unchanged). No
production code change is required.
