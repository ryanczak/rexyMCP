# Bug 2 on phase-02: re-dispatch did not apply the fix from bug-02-1

**Severity:** minor
**Status:** open
**Filed:** 2026-06-16

## What's wrong

The re-dispatch self-reported complete (flipped phase status to `review`)
without making any change to `executor/src/store/telemetry.rs`. Bug-02-1 is
still open: `read_escalations_excludes_review_by_discriminator` remains the
original broken version that was spec-transcribed from the phase doc.

The test body at `executor/src/store/telemetry.rs:1047–1072` appends a
`PhaseReview` record and asserts it is excluded from `read_escalations`.
Because `PhaseReview` lacks `EscalationEvent`'s required fields
`assist_index` / `architect_input_tokens` / `architect_output_tokens`, it
fails to parse as `EscalationEvent` via structural mismatch **before** the
discriminator filter at line 465 is reached. Deleting that filter leaves
both this test and `read_escalations_excludes_run_lines` green — the
discriminator is dead code with respect to the current test suite.

**Mutation proof:** delete `telemetry.rs:465`

```rust
        .filter(|e| e.record == ESCALATION_RECORD_TAG)   // <- delete this
```

Both `read_escalations_excludes_review_by_discriminator` and
`read_escalations_excludes_run_lines` still pass. The discriminator filter
is not covered.

## What should happen

The test must use an **escalation-shaped** line (all required
`EscalationEvent` fields present) with a non-`"escalation"` `record` tag,
so it can only be excluded by the discriminator filter, not by structural
mismatch. See bug-02-1 for the full analysis.

## How to fix

In `executor/src/store/telemetry.rs`, replace the body of
`read_escalations_excludes_review_by_discriminator` (lines 1047–1072) with:

```rust
#[test]
fn read_escalations_excludes_review_by_discriminator() {
    // An escalation-SHAPED line (all required EscalationEvent fields present)
    // with a non-"escalation" record tag must be excluded by the `record`
    // filter — not by structural mismatch. Deleting the
    // `.filter(|e| e.record == ESCALATION_RECORD_TAG)` line in
    // read_escalations MUST fail this test (M18 bug-01-1 lesson: pin the
    // discriminator as load-bearing).
    let dir = TempDir::new().unwrap();
    let mistagged = EscalationEvent {
        record: REVIEW_RECORD_TAG.to_string(), // wrong tag, escalation shape
        ts: 2_000,
        phase_doc_path: Some("/docs/phase-02.md".to_string()),
        phase_id: "phase-02".to_string(),
        project_id: None,
        assist_index: 1,
        model: None,
        architect_input_tokens: 1,
        architect_output_tokens: 1,
    };
    append_escalation(dir.path(), &mistagged).unwrap();
    let path = dir.path().join("phase_runs.jsonl");
    let events = read_escalations(&path).unwrap();
    assert!(
        events.is_empty(),
        "a non-\"escalation\" record tag must be excluded by the discriminator"
    );
}
```

No production-code change is required. This is a test-only fix. The
`read_escalations` discriminator filter at `telemetry.rs:465` is already
correct.
