# Bug 1 on phase-04: scope-escape tests write/read outside the TempDir (non-hermetic)

**Severity:** minor
**Status:** fixed
**Filed:** 2026-05-28

## What's wrong

Both scope-escape tests reach **outside** the `tempfile::TempDir`, into its parent
— the shared system temp root (e.g. `/tmp`). This violates the hermetic-test
standard: STANDARDS §3.3 ("no writes to the host home or shared locations") and
AGENTS.md § Testing ("no host-side state outside a `tempfile::TempDir`").

- `executor/src/tools/patch.rs:495-498`
  (`scope_escape_returns_advisory_error_and_modifies_nothing`) **writes**
  `outside_patch.txt` into `dir.path().parent()` and leaves it there — the
  `TempDir` only cleans `dir.path()`, not its parent, so this litters the temp
  root and can collide across parallel/repeated runs.
- `executor/src/tools/write_file.rs:224-225`
  (`scope_escape_returns_advisory_error_and_writes_nothing`) asserts on
  `dir.path().parent().unwrap().join("outside.txt").exists()` — reading host state
  outside the `TempDir`, so the assertion's correctness depends on whatever else
  happens to be in `/tmp`.

The tool implementations are correct — this is a test-construction issue only. The
escape is exercised by passing `"../outside.txt"`, which is the right behavior to
test; the problem is that the test's *scope root is the `TempDir` itself*, so
"just outside the scope root" lands outside the `TempDir`.

This is partly induced by the phase doc's wording ("stage a file outside the
root") — acknowledged; the fix is to make "outside the scope root" still land
inside the `TempDir`.

## What should happen

The escape test should confine **all** filesystem state to the `TempDir`, while
still pointing the tool at a path that resolves outside the *scope root*. Make the
scope root a subdirectory of the `TempDir`; then a sibling directory inside the
same `TempDir` is "outside the scope root" but still inside the sandbox.

## How to fix

In both `executor/src/tools/patch.rs` and `tools/write_file.rs`, change the
scope-escape tests so the scope root is nested:

```rust
let temp = tempfile::TempDir::new().unwrap();
let root = temp.path().join("root");
std::fs::create_dir(&root).unwrap();
let scope = Scope::new(&root).unwrap();
// "../" from `root` resolves to `temp.path()` — outside the scope root,
// but still inside the TempDir sandbox.
```

- **patch test:** stage the "outside" file at `temp.path().join("outside.txt")`
  (a sibling of `root`, inside the `TempDir`), call `patch` with
  `path: "../outside.txt"`, assert the advisory `ScopeError` and that the file is
  unchanged.
- **write_file test:** call with `path: "../outside.txt"`, assert the advisory
  error and that `temp.path().join("outside.txt")` was not created.

Keep asserting the error is advisory (mentions escaping the root) and that no
write/modification occurred. No production-code changes; no new dependencies.
