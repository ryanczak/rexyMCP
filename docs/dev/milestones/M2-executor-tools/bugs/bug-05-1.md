# Bug 1 on phase-05: classifier over-blocks benign commands via bare-substring matching

**Severity:** minor
**Status:** verified
**Filed:** 2026-05-28
**Verified:** 2026-05-28 — `293edf1`. System-control / privilege / kill commands
moved to a command-position regex (`DANGEROUS_CMD_RE`, matches at start or after
`;&|(`/newline); removed from the substring list, `init 0/6` folded in. New tests
confirm `cargo test shutdown` / `grep -rn shutdown` / `grep asphalt` → `Allow`
while `shutdown now` / `ls; reboot` / `(halt)` → `Block`. 165 tests, gates clean.

## What's wrong

`bash_classify::classify` matches several system-control commands as **bare
substrings** of the normalized command (`executor/src/security/bash_classify.rs`,
`BLOCK_SUBSTRINGS` at lines 58-77). Because these are common English words, they
match inside perfectly benign commands and refuse them:

- `"shutdown"` → `classify("cargo test shutdown")`, `classify("grep -rn shutdown
  src/")`, `classify("./scripts/shutdown_test.sh")` all return `Block`.
- `"halt"` → matches `asphalt`, `cobalt`, a module/test named `halt`
  (`classify("grep halt notes.txt")` → `Block`).
- `"reboot"`, `"poweroff"` → same class of false positive.

A coding executor frequently runs things like `cargo test shutdown` or
`grep -rn shutdown` while working on server/lifecycle code; silently refusing them
(and a weak model won't understand *why*) will derail phases. The classifier errs
safe security-wise, but blocking legitimate work is itself a correctness defect.

This is partly induced by the phase-05 spec: Pre-injection §A listed these as plain
substrings and the pinned tests only asserted the *positive* cases
(`classify("shutdown now") == Block`), never the benign-substring case. The
implementation faithfully realized the spec — the spec was under-specified.

## What should happen

The system-control commands should match only when they appear as the **command
being invoked** — at the start of the command or immediately after a shell
separator (`;`, `&&`, `||`, `|`, newline, `(`) — not as an argument or a substring
of another word. `shutdown now`, `foo && reboot`, `; poweroff` must still
`Block`; `cargo test shutdown`, `grep halt`, `grep -rn shutdown src/` must
`Allow`.

## How to fix

In `executor/src/security/bash_classify.rs`, move the bare system-control words out
of `BLOCK_SUBSTRINGS` into a command-position regex (compiled once via
`LazyLock`), e.g.:

```rust
static SYS_CONTROL_RE: LazyLock<Regex> = LazyLock::new(|| {
    Regex::new(r"(?:^|[;&|(\n])\s*(?:shutdown|reboot|halt|poweroff|init\s+[06])\b").unwrap()
});
```

- Apply it to the already-normalized string (the `init 0` / `init 6` regexes then
  fold into this one).
- Keep the genuinely-rare-as-text commands as substrings if you prefer, but
  consider the same command-position treatment for `sudo ` and `su ` (e.g.
  `echo "run with sudo"` currently → `Block`); `kill -9`, `pkill`, `killall`,
  `git push`, and the publish commands are low-risk as substrings and may stay.
- Secondary (optional): `CHOWN_R_ROOT_RE` (`chown -r `) blocks *any* recursive
  chown, not just on `/` — narrow it to root/absolute targets if convenient, but
  this is not required to close the bug.

## Verification

- [ ] `classify("cargo test shutdown")`, `classify("grep -rn shutdown src/")`, and
      `classify("grep halt notes.txt")` return `Severity::Allow`.
- [ ] `classify("shutdown now")`, `classify("reboot")`, `classify("foo && reboot")`,
      `classify("init 0")`, `classify("init 6")` still return `Severity::Block`.
- [ ] New tests cover both the benign-substring (Allow) and command-position
      (Block) cases for the system-control words.
- [ ] `cargo fmt --all --check`, `cargo build`, `cargo clippy --all-targets
      --all-features -- -D warnings`, `cargo test` all pass clean.

## Notes

Everything else in phase-05 is strong: the classifier gate is wired before spawn,
cwd-pin and env-strip are correct, the env predicate is unit-tested without
mutating process env, all four Rexy TODOs were dropped, and the blocked-not-
executed test is exactly the right shape. This is a targeted false-positive fix on
the curated list, not a structural problem.
